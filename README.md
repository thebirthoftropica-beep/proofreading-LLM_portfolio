# 記者ハンドブック準拠・校閲専用LLM（Proofreading LLM）

『記者ハンドブック（共同通信社）』の規範を学習させた、報道・実務向けの文章校閲AIモデルです。

## 🚀 プロジェクトの目的
汎用的なAIでは見落とされがちな、新聞業界特有の表記ルールや誤用を正確に検知・修正することを目指しています。
（例：慣用句の誤用「雪辱を晴らす」→「雪辱を果たす」への修正など）

## 🛠 使用技術
- **Base Model:** `elyza/Llama-3-ELYZA-JP-8b`
- **Fine-tuning:** Unsloth / LoRA (PEFT)
- **Environment:** Google Colab
- **Library:** `transformers`, `trl`, `peft`, `bitsandbytes`

## 📘 実装のポイント
本プロジェクトでは、限られた計算資源で高精度な校閲を実現するため、以下の手法を採用しています。
- 本業で構築したローカルLLMの**模擬・ミニ版**です。
- **4bit量子化:** メモリ消費を抑えつつ、推論精度を維持。
- **LoRA:** モデル全体の重みではなく、校閲タスクに必要な「差分」のみを効率的に学習。
- **Alpaca形式プロンプト:** 指示・入力・出力を論理的に分離し、AIの回答精度を向上。

## 💻 実行方法
このプロジェクトは Google Colab で動作を確認しています。

1. リポジトリ内の `proofreading.ipynb` を開く
2. `Open In Colab` バッジをクリックして実行
3. `data.csv`（学習データ）をアップロードして学習開始
